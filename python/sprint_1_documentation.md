### 3. Python Concepts Covered
Here is a breakdown of the core concepts I explored during this sprint:

* **Python Basics & Syntax:** Python uses indentation instead of curly braces to define code blocks, making it highly readable. Comments are created using the `#` symbol.
* **Variables and Data Types:** Variables act as storage containers for data. I worked with Integers (`int`), Floats (`float`), Strings (`str`), and Booleans (`bool`). Understanding data types is crucial because it dictates what operations can be performed on that data.
* **Operators:** I learned how to manipulate data using:
    * *Arithmetic:* For math operations (`+`, `-`, `*`, `/`, `%`)
    * *Comparison:* To compare values, returning True/False (`==`, `>`, `<=`)
    * *Logical:* To combine conditions (`and`, `or`, `not`)
    * *Assignment:* To assign and update values (`=`, `+=`)
    * *Membership:* To check if a value exists in a sequence (`in`, `not in`)
* **Working with Strings:** Strings can be wrapped in single, double, or triple quotes. I practiced concatenating strings with `+`, slicing them using zero-based indexing (e.g., `string[0:5]`), and formatting them beautifully using f-strings.
* **Conditionals:** Using `if`, `elif`, and `else` statements allows the program to make decisions and execute different blocks of code based on specific conditions.
* **Loops:** I used `for` loops to iterate over sequences (like lists or `range()`) and `while` loops to repeat code as long as a condition remains true. I also learned to control these loops using `break` and `continue`.
* **Functions:** Functions allow code to be grouped into reusable blocks. By using parameters, functions can accept inputs, and by using `return`, they can pass computed values back to the rest of the program.
